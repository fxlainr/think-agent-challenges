## Vision & Impact

Utiliser des frameworks de prompting pour l'**idéation de design**. L'objectif est de briser le syndrome de la page blanche en générant des concepts variés en quelques secondes.

## Le saviez-vous ?

NotebookLM peut être couplé à Figjam pour transformer vos notes de recherche en flux d'utilisateurs visuels. L'IA ne remplace pas la créativité, elle l'amplifie.

## Description

> *Utilisez les frameworks de prompting pour générer des dizaines d'idées de design en quelques minutes.*

Page blanche. Cursor clignotant. Deadline dans 2 jours. Ce challenge vous donne les frameworks pour débloquer votre créativité avec l'IA et explorer des directions que vous n'auriez jamais envisagées seul.

---

## Objectif du Challenge

**Ce que le participant doit produire :**

- [ ] 20+ concepts générés avec 3 frameworks différents
- [ ] 5 concepts sélectionnés et enrichis
- [ ] 1 concept développé en wireframe

**Critères de réussite :**

- Variété des directions explorées
- Au moins 1 idée "inattendue" dans le lot
- Le concept final est argumenté

## Pistes de Solution

### Framework 1 : SCAMPER

```
Applique SCAMPER à [concept de base] :

S - Substituer : Que peut-on remplacer ?
C - Combiner : Que peut-on fusionner ?
A - Adapter : Que peut-on emprunter ailleurs ?
M - Modifier : Que peut-on amplifier ou réduire ?
P - Put to other uses : Autres usages possibles ?
E - Éliminer : Que peut-on supprimer ?
R - Réorganiser : Que peut-on inverser ?

Génère 2-3 idées par lettre.
```

### Framework 2 : Analogies forcées

```
Je conçois [type d'interface/produit].

Génère des idées en utilisant ces analogies :
1. Si c'était un restaurant 5 étoiles ?
2. Si c'était un jeu vidéo ?
3. Si c'était conçu pour des enfants de 5 ans ?
4. Si c'était conçu pour des astronautes ?
5. Si c'était un produit Apple des années 2030 ?

Pour chaque analogie, donne 3 insights applicables.
```

### Framework 3 : Contraintes extrêmes

```
Concept : [votre concept]

Génère des variations avec ces contraintes :
1. Budget : 0€, tout doit être gratuit
2. Temps : L'utilisateur a 10 secondes max
3. Écran : Interface uniquement vocale
4. Contexte : Utilisé en marchant dans la rue
5. Utilisateur : Personne de 80 ans non-technophile

Comment le concept s'adapte à chaque contrainte ?
```

### Framework 4 : "Et si...?"

```
Concept de base : [votre concept]

Explore ces "Et si...?" :
- Et si c'était l'inverse ? (l'utilisateur devient le produit)
- Et si c'était 10x plus simple ?
- Et si c'était 10x plus complexe mais magique ?
- Et si ça n'existait pas ? Comment les gens feraient ?
- Et si c'était illégal ? Pourquoi ?
- Et si c'était gratuit pour toujours ?
```

### Prompt de synthèse

```
Voici 20 idées générées pour [projet] :
[coller les idées]

Analyse-les et :
1. Identifie les 3 patterns qui reviennent
2. Trouve l'idée la plus originale
3. Trouve l'idée la plus faisable
4. Propose 1 hybride combinant le meilleur de plusieurs idées
```

## Animation Dojo (1h, ~10 personnes)

### Objectifs pédagogiques

À la fin de l'atelier, les participants sauront :

- Utiliser 3+ frameworks d'idéation avec l'IA
- Générer rapidement des concepts variés
- Sélectionner et enrichir les meilleures idées

### Déroulé minute par minute

| Temps | Phase | Activité |
|-------|-------|----------|
| 0-5 | Intro | Le syndrome de la page blanche |
| 5-15 | Démo | SCAMPER en live sur un cas |
| 15-30 | Sprint 1 | Chacun génère 20 idées (3 frameworks) |
| 30-40 | Sélection | Identifier les 3 meilleures |
| 40-50 | Partage | Tour de table des idées les plus folles |
| 50-60 | Clôture | Comment intégrer ça dans son workflow |

### Points d'attention animateur

- Encourager les idées "stupides" — souvent les meilleures
- Imposer la quantité avant la qualité
- Chronométrer pour garder le rythme
