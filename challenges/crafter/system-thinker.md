## Vision & Impact

Adopter une pensée systémique pour **zoomer et dézoomer** sur les problèmes complexes. L'IA permet de rendre le design plus significatif en gérant la complexité des systèmes.

## Le saviez-vous ?

La pensée systémique n'est plus un plafond, mais le **nouveau socle** de notre métier. Ceux qui ne pensent qu'en features sont déjà obsolètes.

## Description

> *Apprenez à voir les systèmes derrière les features.*

Un bouton n'est jamais juste un bouton. C'est un point dans un système d'utilisateurs, de flux, d'incentives. Ce challenge vous fait développer votre capacité à voir et designer des systèmes, pas des écrans.

---

## Objectif du Challenge

**Ce que le participant doit produire :**

- [ ] Cartographie systémique d'un produit existant
- [ ] Identification des boucles de feedback
- [ ] Proposition d'intervention sur les leviers clés

**Critères de réussite :**

- Vision macro ET micro articulée
- Boucles de renforcement/équilibrage identifiées
- Interventions ciblées sur les leviers

## Pistes de Solution

### Les concepts clés

```
SYSTÈME = Éléments + Interconnexions + Fonction

BOUCLE DE RENFORCEMENT (R)
Plus de X → Plus de Y → Plus de X
Ex: Plus d'utilisateurs → Plus de contenu → Plus d'utilisateurs

BOUCLE D'ÉQUILIBRAGE (B)
Plus de X → Plus de Y → Moins de X
Ex: Plus de croissance → Plus de bugs → Moins de croissance

DÉLAI
L'effet n'est pas immédiat
Ex: Investir en brand → [6 mois] → Croissance organique

LEVIER
Point où une petite action crée un grand effet
```

### Prompt d'analyse systémique

```
Analyse ce produit/situation avec une pensée systémique :

Produit : [X]

Cartographie :
1. Quels sont les éléments clés du système ?
2. Comment sont-ils interconnectés ?
3. Quelle est la fonction/but du système ?

Boucles :
4. Identifie 2-3 boucles de renforcement
5. Identifie 2-3 boucles d'équilibrage
6. Où sont les délais importants ?

Leviers :
7. Où une petite action aurait un grand impact ?
8. Quels sont les effets secondaires potentiels ?

Visualise avec un diagramme causal.
```

### Exercice : Cartographier Uber

```
Éléments : Chauffeurs, Passagers, Prix, Temps d'attente, App

Boucles de renforcement :
- Plus de chauffeurs → Moins d'attente → Plus de passagers → Plus de revenus → Plus de chauffeurs

Boucles d'équilibrage :
- Plus de demande → Prix surge → Moins de demande

Leviers :
- Le pricing dynamique = levier majeur du système

Délais :
- Recrutement chauffeurs = semaines
- Changement de comportement passagers = immédiat
```

### Niveaux de zoom

```
ZOOM OUT (Macro)
├── Écosystème (marché, concurrents, régulation)
├── Organisation (équipes, processus, culture)
└── Produit (vision, stratégie, roadmap)

ZOOM IN (Micro)
├── Feature (user story, specs)
├── Composant (UI, code)
└── Interaction (micro-copy, animation)

L'ART = Naviguer fluidement entre les niveaux
```
